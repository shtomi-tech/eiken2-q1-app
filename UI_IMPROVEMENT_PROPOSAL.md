# UI改善案 — ヒック法則・フィッツの法則・ゲシュタルト原則

作成日: 2026-07-14
対象: `index.html` / `static/app.js` / `static/mode-q1.js` / `static/mode-q3.js` / `static/styles.css`

前提となる現状評価:
- 大問1ホームは既にヒック法則対応済み（`mode-q1.js` の「おすすめ＝主導線1つ」パターン、`recommend` ブロック）。
- 破壊的操作（進捗リセット）は既に `<details>「その他」` に隔離済み（フィッツ的に正しい配置）。
- 全ボタンに `min-height: 44px` の下限あり（一部例外は後述）。

以下は**未対応・不徹底の箇所**に絞った改善案。優先度は ★★★（効果大・工数小）〜★（任意）。

---

## 1. ヒックの法則（選択肢を減らし、最初の行動を明確に）

### 1-1. ★★★ 大問3ホームにも「▶ まずはここから」主導線を追加
**現状**: `mode-q3.js renderHome()` は、復習バナー＋文章カード×N（各カードに「設問演習」「内容整理」の2ボタン）が同格に並び、初見の生徒はどれを押すべきか判断が必要。文章3本なら同格の選択肢が最大7個。
**案**: 大問1と同じ `recommend` パターンを移植し、状態に応じて主導線を1つだけ表示する。優先順位:
1. 復習対象あり → 「間違えたN件をまとめて復習する」（既存 `startReview`）
2. 未完了の文章あり → 「3A『（タイトル）』の続きから解く（3/8問）」（既存 `openPractice`）
3. 設問全問正解・内容整理未了 → 「内容整理（要約穴埋め）に進む」
4. 全完了 → 「もう一周する」

「なぜこれか」の1行（`recWhy` 相当）も添える。文章カードは従属メニューとしてそのまま残す。

### 1-2. ★★★ 大問1ホームの重複カードを統合（Today と Mission）
**現状**: 「Today」カード（学習した設問／本番で正解／間違えた設問）と「Mission」カード（通常ステージCLEAR／復習対象／最終チェックBEST）で、6セル中4セルが実質同じ情報の言い換え。読む量と判断対象が倍になっている。
**案**: 1カードに統合し、statセルは「学習済 n/N」「正解 n/N」「復習対象 n/N」「最終チェックBEST n/M」の4つ（モバイルは2×2）。`finalMessage()` の解放条件テキストはこの統合カード内に残す。

### 1-3. ★★ hero カードと Today 見出しの説明重複を解消
**現状**: hero「大問1の語句を『覚えてから解く』…3ステップ」と Today 見出し「覚えて→確かめて→解く」がほぼ同文。
**案**: hero は初回訪問時のみ表示（`progress.units` が空のときだけ）にするか、Today 見出しを回ラベル＋データセット選択だけの1行に簡素化する。

### 1-4. ★★ 迷いやすいラベルの具体化
- 大問3「設問演習」→ 進捗を含めて「設問を解く（3/8問 解答済み）」。未着手なら「設問を解く（全8問）」。
- 大問1 従属メニュー「最終チェック（あとN問で解放）」は良い。同様に大問3の「内容整理」も採点済みなら「内容整理（5/6 正解・やり直し）」と状態を出す。
- アプリタブ（`app.js APPS`）: 「大問1（語彙）」「大問3（長文）」に加えて、タブの `tag` を "VOCAB QUIZ"/"READING" の英字ラベルでなく「単語を覚える」「長文を読む」にすると初心者の判断が速い（現代文・英語の生徒対象なら英字でも可。任意）。

## 2. フィッツの法則（重要な操作ほど大きく・近く・誤タップしにくく）

### 2-1. ★★★ 極小タッチターゲットの解消
| 対象 | 現状 | 案 |
|---|---|---|
| `.smallGhost`（大問3「一覧に戻る」、誤答復習「確認した」） | min-height 34px | 44px に引き上げ。見た目を小さく保ちたいなら `padding` を透明のヒット領域として確保 |
| `.linkBtn`（和訳の表示/非表示） | min-height:auto・padding 0・11px 文字 | `padding: 12px 0` を追加し実効ヒット高 ≥44px に。文字サイズは据え置き可 |
| `.q3Session .blank`（要約の空所） | min-width 90px・行内 span | `min-height: 32px` + `padding: 4px` を追加し `line-height` を確保（行間 1.9 で救われているが指では狭い） |

### 2-2. ★★★ 「前へ」「次へ」の間隔と重み付け
**現状**: 暗記カードの「← 前のカード」「次のカード →」、大問3の「前の設問」「次の設問へ」が gap 8〜10px で隣接。逆方向の操作が近すぎ、誤タップすると1枚戻る/進むが混ざる。
**案**:
- 「次へ」（主操作）を大きく: モバイルでは幅 100%、その下ではなく**左端に「前へ」を小さく分離**（`justify-content: space-between` で両端配置、gap ≥24px）。
- 大問1フラッシュカードは既に `FLASH_NAV_GUARD_MS` の連打ガードあり。配置分離と併用でほぼ解消。

### 2-3. ★★ 回答後の「次へ」を親指ゾーンに固定（モバイル）
**現状**: 回答後 `revealAnswerActions()` で「次へ」までスクロールしてくれるが、ボタン自体はフィードバック文の下に流れる位置。連続演習では毎回スクロール＋照準が必要。
**案**: セッション画面のみ、回答後の主ボタンを画面下部に固定するスティッキーバー（`position: sticky; bottom: 0`、背景 `--parchment`・上罫線1px）にする。誤タップ防止に選択肢群との距離が常に確保され、連続演習のテンポも上がる。※デザイン言語（影ゼロ・罫線区切り）と両立可。

### 2-4. ★★ 大問3「スキップ」と「この根拠で答え合わせ」の分離
**現状**: `navRow` 内で隣接（gap 10px）。スキップ＝根拠記録なし、は取り返しがつかない（その設問の根拠一致が "miss" 相当で確定）。
**案**: スキップを左端の ghost・小、答え合わせを右端の大ボタンにし、両端配置で物理的に離す。

### 2-5. ★ 選択肢ボタンの間隔
**現状**: `.choices` gap 10px（大問1）/8px（大問3）。52px 高のボタンに対し 8px はモバイルでやや窮屈。
**案**: gap を 12px に統一（スペーシングスケール 4/8/12/16/24/32 にも整合）。

## 3. ゲシュタルト原則（近接・類同・閉合で構造を見せる）

### 3-1. ★★★ 同種要素の見た目統一（類同）— 選択肢ボタンと正誤表示
**現状**: 同じ「4択の選択肢」なのに、
- 大問1: hover で黒反転、正解＝緑枠2px（背景変化なし）
- 大問3: hover で薄ベージュ、正解＝緑背景＋緑枠

と2流儀ある。番号バッジも `.key`（枠付き正方形）と `.num`(素の文字) で異なる。
**案**: どちらかに寄せる。推奨は**大問1の流儀**（枠のみで示す方がモノクロ印刷的なデザイン言語に合う）を基準に、大問3の `.q3Session .choiceBtn` の背景塗りを外し、`.num` を `.key` と同じ枠付きバッジにする。正誤フィードバック（大問1 `.feedback` の左太罫カード / 大問3 `.resultBox` のタグ行）も、「結果タグ＋解説」の同一フォーマットに統一する。

### 3-2. ★★ 大問3ホームの文章カード内部のグルーピング（近接）
**現状**: `passageCard` 内で「タイトル」「統計行」「ボタン行」が同一余白で縦積みされ、統計がタイトルの説明なのか操作の説明なのか曖昧。
**案**:
- タイトルブロック（psub＋ptitle）と統計行の間は 12px、統計行とボタン行の間は罫線（`border-top: 1px solid var(--line)`）＋16px で分離し、「情報ゾーン／操作ゾーン」を閉合で分ける。
- 統計は「設問」「内容整理」の2列ミニグリッドにし、それぞれのボタンの真上に対応する統計が来るよう揃える（近接による対応付け）。

### 3-3. ★★ ホーム全体のセクション見出しルール統一
**現状**: 大問1ホームは `label`（mono大文字）＋`h2` のセクションヘッダで統一されているが、大問3ホームは hero 以外に見出しがなく、復習バナー・文章リストが「どの区画か」不明。
**案**: 大問3にも「Review」「Passages」等の `label` ＋ `h2` を付け、カード（`section.card`）で括って両モードの画面文法を揃える。モード切替時に同じ構造が現れることで学習コストが下がる（類同・連続性）。

### 3-4. ★ `.cta` の角丸を除去
**現状**: `.cta { border-radius: 0 0 8px 8px }` だけ下角が丸く、他要素の角丸 0px と不整合（デザイン指針「角丸0px」にも反する）。
**案**: `border-radius: 0` に統一。強調は既存どおり反転＋太字で足りる。

### 3-5. ★ 余白スケールの微調整
**現状**: 14px / 18px / 10px などスケール外の値が散在。
**案**: 新規・変更箇所から順に 4/8/12/16/24/32 に寄せる（一括置換はしない。Surgical Changes 原則）。
