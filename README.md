# 英検 演習（大問1・大問3・リスニング・英作文・準1級過去問）

英検2級・準2級の技能別演習と、準1級の過去問3回分を1つの静的Webアプリにまとめたものです。

- **級を選ぶ**タブ：最初に2級・準2級・準1級を選び、その級の学習ルートまたは自由演習へ進みます。選んだ級はブラウザに保存され、各技能の問題データも同じ級に揃います。
- **学習ルート**タブ：大問1（語彙）→言い換え（補助）→英作文→リスニング→大問3（長文）の順に進みます。大問1・英作文・リスニング・大問3は過去問3回分、言い換えは共通8問の補助練習を完了すると次を解放します。各モードの既存進捗を読み取り、回ごとの途中状態も引き継ぎます。
- **自由演習**タブ：順番のロックなしで、五つの技能からその日に取り組むものを選べます。各技能の回答・下書き・途中位置は既存の保存領域を使います。
- **準1級モード**タブ：2026年度第1回・2025年度第3回・2025年度第2回を切り替え、大問1〜3・リスニング・ライティングを回ごとに演習します。選択問題の回答とライティングの確認済み状態は、回別に保存されます。

- **大問1（語彙）**タブ：英検2級・準2級の各回を切り替え、選択肢の意味・語源・例文を確認 → 意味チェック → 本番形式の4択、の3ステップで演習。誤答復習・最終チェックあり。
- **大問3（長文）**タブ：4択解答 → 根拠だと思う文を本文からタップ選択 → 判定（選択肢正誤＋根拠一致度）。内容整理（要約穴埋め）と誤答・根拠不一致の復習モードあり。
- **リスニング**タブ：級・回セレクタで設問ごとの音声を再生 → 4択回答 → 誤答のみ書き取り・スクリプト確認、の3ステップで演習。
- **言い換え練習**タブ：英単語を思い出せないとき、もの・人・場所を関係詞で説明する練習。英検準2級レベルを意識した基本語の例答・日本語解説・チュートリアルを収録。
- **英作文**タブ：テーマの英文を訳す → HEADでYes / Noを選ぶ → BODY 1・2の理由を日本語で考えて平易にする → CONCLUSIONを書く → 全体をレビューする、の6段階で演習。2級は80–100語、準2級は50–60語を全体の目安として表示します（文字数ではなく英語の語数）。

各モードは `static/mode-*.js` に IIFE で分離されており、`static/app.js` が級入口・学習ルート・自由演習・準1級モードと各技能モードを切り替えます（kobun-vocab と同じ方式）。進捗の保存キー（localStorage）はモードごとに別々で、学習ルートは主要4パートの回別進捗を集計して次の段階を解放します。統合前の進捗もそのまま引き継がれます。

## 学習の流れ（大問1）

1. 選択肢の意味・語源・例文を確認する
2. 意味チェックを行う
3. 本番形式の4択問題を解く

通常の進捗はブラウザのローカルストレージに保存されます。

## 起動

```powershell
cd C:\Users\shtom\dev\eiken-practice
py -3 -m http.server 8061 --bind 127.0.0.1
```

ブラウザで `http://127.0.0.1:8061/` を開きます。データJSONは相対パスで読み込むため、`index.html` を直接開かずHTTPサーバー経由で確認してください。

## 公開版・生徒別進捗

公開版は [GitHub Pages](https://shtomi-tech.github.io/eiken2-q1-app/) です。

生徒別URLの `?s=<id>&t=<token>` が付く場合は、公開設定が有効なら共通Supabaseスキーマの `app_students` / `app_progress` へ進捗を同期します（大問1・大問3ともappIdが異なる行として保存されます）。設定がない通常利用では、従来どおりローカル保存だけで動作します。生徒の登録SQLとQR発行は `portal` が担当します。

## データと共通エンジン

- 大問1のデータ: `data/questions_*.json` と `data/vocab_*.json`
- 大問3のデータ: `data/q3_questions_2026-1.json`
- リスニングのデータ: `data/lessons*.json`（設問データ）と `assets/clips/` `assets/audio/`（音声。約240MB）
- 準1級過去問のデータ: `data/pre1_*.json`。公式PDF・音声をローカルで構造化するスクリプトは `scripts/build_pre1_data.py`。元ファイルの `data/eiken_p1/` は公開用データとして扱わない。
- 画面・アプリ固有ロジック: `static/mode-entry.js`（級入口）・`static/mode-serial.js`（学習ルート）・`static/mode-free.js`（自由演習）・`static/mode-pre1.js`（準1級過去問）・`static/mode-q1.js`（大問1）・`static/mode-q3.js`（大問3）・`static/mode-dictation.js`（リスニング）・`static/mode-paraphrase.js`（言い換え練習）・`static/mode-writing.js`（英作文）・`static/app.js`（画面切替の薄いシェル）
- 言い換え練習データ: `data/paraphrase_questions.json`（既存の大問データ・進捗キーとは分離）
- 英作文データ: `data/writing_questions.json`（下書き・レビュー済み状態も既存モードとは分離）
- 共通クラウド同期層: `static/vendor/harness/`

`static/vendor/harness/` は `C:\Users\shtom\dev\learning-engine` から配布される生成物です。直接編集せず、更新後は次で確認します。

```powershell
cd C:\Users\shtom\dev\learning-engine
node scripts/sync.mjs --check
```

`static/config.json` には公開時の接続設定が入るためgit管理しません。ローカル確認用には `static/config.example.json` を参照してください。

## 由来

このリポジトリは、もともと大問1（語彙）のみだった `eiken2-q1` に `eiken2-q3`（大問3・長文読解）、続いて `eiken-dictation`（リスニング・ディクテーション）を統合したものです。公開URLとQRコード配布・Supabase生徒別クラウド同期を維持するため、器はこちら（eiken2-q1）側を採用しています（`kobun-vocab` が `kobun-katsuyo` を吸収したのと同じ方式）。旧 `eiken2-q3-app`（GitHub Pages）は本アプリへの転送ページに差し替え、リポジトリはアーカイブ済みです。旧 `eiken-dictation` も同様に転送ページ化してアーカイブする予定です。
